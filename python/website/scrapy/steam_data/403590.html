Early Access Game

Get instant access and start playing; get involved with this game as it develops.

Note: This Early Access game is not complete and may or may not change further. If you are not excited to play this game in its current state, then you should wait to see if the game progresses further in development. Learn more

What the developers have to say:

Why Early Access?

“This game is still under heavy development. Players should expect bugs and missing features. New features, optimization and bug fixes will be added all throughout early access.

We've setup a great foundation for this game to grow on and while there's still a solid year of development ahead of us, Early Access allows us to grow our community with the game. Their feedback helps us ensure we are making a game they want to play rather than just a game we assume they will enjoy.

As an added bonus, releasing the game while so early in development also allows us to invest the sales directly into the game to make it a far better game than our personal credit cards can hope to achieve.”

Approximately how long will this game be in Early Access?

“We estimate a year of development still remains with our current resources. However, if funds allow us to grow our team we can greatly reduce that timeline.”

How is the full version planned to differ from the Early Access version?

“The full version of Klepto is planned to include additional neighborhoods, multiplayer game modes, tools, disguises, missions, contacts and the single player story. Visit our Trello board for the full Early Access Roadmap.”

What is the current state of the Early Access version?

“The current state of Early Access includes all the core framework for the game along with 2 sandbox neighborhoods, co-op multiplayer game mode, mission system and full VR support.”

Will the game be priced differently during and after Early Access?

“As an added incentive for supporting the game early and because we are launching as Early Access with a much smaller scope than initially planned, we have made a significant price reduction for the time being.

We will likely have one price increase at some point during Early Access but before we do so, we will make announcements of the planned change at least a full month in advance. There will likely be a final price increase following Early Access when the game is in a completed state.”

How are you planning on involving the Community in your development process?

“Community feedback is one of our primary reasons for releasing with Early Access. In addition to regular Q&A, we've setup a Trello board for our Early Access roadmap and we encourage everyone to participate by commenting and casting votes for the features they're anticipating the most.

Comment and vote on the full list of upcoming features at https://trello.com/b/bw8DJ0C7/klepto-early-access-roadmap”

Alpha 0.1.1 Release Notes



Build: Alpha 0.1.1
Date: 07.06.16

Alpha 0.1.1 is now available! The focus of this week’s patch has primarily been bug fixes. Please continue to report new issues HERE.

Main Menu

  • Added Options Menu to main menu desktop. This will be added to tablet in 0.1.2 update (KBT-21)
  • Added notification on level select screen that Fragbound Farms is in a preview state. (KBT-24)
  • Virtual mouse cursor now defaults to visible location on the klept_os screen. (KBT-37)

Neighborhoods
  • New residents added to Murky Meadows
  • Adjusted 1st-person camera so neck isn't visible while sprinting
  • Improved AI pathing for cops (KBT-28)
  • Optimized streaming volumes used to load house interiors (KBT-19) - More improvements coming in Alpha 0.1.2.
  • Houses no longer appear empty while interiors are being loaded. (KBT-19)
  • 2-Hand Carry items no longer despawn when house interior offloads (KBT-15)
  • Collision improvements for various walkways and steps (KBT-17)
  • Removed driving sequence in Oldtown (KBT-34)

Tutorial Level
  • Added Out of Order sign to elevator in Tutorial until feature is enabled. (KBT-23)
  • Fixed issue causing game crash in tutorial level when tablet was opened. (KBT-18)

Tablet
  • Added Home Page help info when tablet is opened with Gamepad (KBT-32)
  • Updated keyboard shortcuts (see Controls section of the Release Notes)
  • Mouse cursor now auto-toggles when keyboard is used to open tablet (KBT-33)

VR
  • Added VR specific game options to main menu (KBT-21)
  • Changed default VR turn method to controller yaw. Configurable from new options menu.
  • Motion controllers root position now rotates with controller yaw (KBT-25)
  • Added auto lock picking for motion controllers. Full motion support coming in future update. (KBT-26)
  • Added basic motion control support for driving. Usage is same as gamepad. (KBT-27)

Controls
  • Interact (“E” Key or A Button) now automatically equips required tool (KBT-31)
  • Added keyboard layout image to tutorial and main menu (KBT-11)
  • Added Invert Y option (KBT-21)
  • Changed Chat keybinding to “/” Key
  • Changed Contracts keybinding to “C” Key
  • Increased interaction distance (KBT-16)
  • Disabled ability to attack in Oldtown which triggered fade to black issue. (KBT-14)
  • Disabled tool usage / attack when tablet is open. (KBT-29)

Multiplayer
  • Minor Replication / game sync improvements (KBT-13)

Known Issues
  • KBT-00: Game is buggy and everyone hates it. Fix in progress.
  • KBT-36: Improvements to safe cracking combo reset triggers. Fix coming in 0.1.2 next week.
  • KBT-41: Loot value placeholders missing for some categories. Fix coming in 0.1.2 next week.
  • KBT-42: CryptoKoin (cK) values no persistent. Fix coming in 0.1.2 next week.
  • KBT-22: cK values no updated when contracts are completed. Fix coming in 0.1.2 next week.
  • KBT-40: (New Feature) Tablet support for new Options menu coming in 0.1.2 next week.
  • KBT-39: (New Feature) Sell All” option for Inventory coming in 0.1.2 next week.
  • KBT-43: (New Feature) Input key remapping system coming in 0.1.2 next week.
  • KBT-48: Loading screen progress wheel stops spinning before loading is completed.
  • KBT-49: Main Menu virtual cursor sticks to edges when mouse is moved too far right or down.
  • KBT-44: Game always launches in VR mode when HMD is detected.
  • KBT-10: Main Menu virtual cursor not working with in-home streaming.
  • KBT-12: High CPU utilization reported by some users
  • KBT-9: Van shaking when driving - shocks need to be replaced?
  • KBT-13: Multiplayer replication / game sync issues. Improved in 0.1.1. Additional work in progress.

About This Game

Welcome to Klepto where there's no place like a burglarized home. In this burglary simulation you'll explore, terrorize and loot your way to success in a variety of open-world sandbox neighborhoods.

Build your reputation in the criminal world by taking on contracts or team up with your fellow criminals for some online multiplayer shenanigans. Hate the idea of burglarizing? Play online as one of Klepto's finest to serve and protect the neighborhoods in the Cops & Robbers multiplayer game mode.

**VR is NOT required for this game.


  • Sandbox Mode (Single Player)
  • Multiplayer Co-Op
  • 2 Open-world neighborhoods
  • Full VR Support
  • Lockpicking & Safe Cracking
  • Powered by Unreal Engine


  • Story Mode (Single Player)
  • 4 New Neighborhoods
  • Residential Hacking featuring popular tools like NMAP and the Aircrack-ng Suite
  • Disguise System
  • Additional Multiplayer Game Modes
  • Multiplayer Enhancements including Progression and Reputation Systems
  • More gadgets, tools and hacks
