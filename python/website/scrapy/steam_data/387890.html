About This Game

Radial Impact is a fast-paced, single-player game that challenges players’ reflexes by matching the sizes of two circles; one static and one dynamic.

Game Features:

Single-Player.
Multiple Game Modes (Classic, Time Trial, and Waves).
Simple Mechanics (Click to Interact).
Dynamic Circle Sizes and Locations.
Developed solely by Devin Sherry.

Classic Mode – Overview

Game-play focused on getting the highest score.
Points earned based on how close the player matches the shape of the circles.
Three Lives – Once all lives are lost, the player loses and the game ends.
Slow-down ability unlocked if the player gets a Perfect rating 5x in a row.
If the player reaches a 10X Perfect multiplier, they earn back a life if one was lost.
Lives are lost if the player gets either a Bad or a Miss rating.
Saved Data for High Score.

https://www.youtube.com/watch?v=TptSCxxTkos

Time Trial – Overview

Game-play focused on lasting the longest without time running out.
Time is added and subtracted based on how close the player matches the shape of the circles.
The player starts with 59 seconds until the game ends.
Slow-down ability unlocked if the player gets a Perfect rating 5x in a row.
Saved Data for Longest Time.

https://www.youtube.com/watch?v=BwH2Q-p2m9s

Waves Mode – Overview

The goal of the player is to earn a score that is larger than the “Score to Beat” for each waves option and set there own high score.
There are multiple “Waves” options for the player to choose from. There are 5 Waves, 10 Waves, 25 Waves, and 50 Waves.
A “Wave” consists of one turn of the player matching the two circles shapes.
At the start, only the “5 Waves” mode is unlocked for the player. As the player scores higher than the “Score to Beat” the player will unlock the higher waves options.
Slow-down ability unlocked if the player gets a Perfect rating 5x in a row.
Saved Data to track which waves options have been unlocked.

https://www.youtube.com/watch?v=zs3Na0Vg0x8

System Requirements

    Minimum:
    • OS: Microsoft Windows 7 or higher
    • Processor: 1.80GHz Processor
    • Memory: 2 GB RAM
    • Graphics: Video card with 512MB of VRAM or better
    • DirectX: Version 9.0
    • Storage: 150 MB available space
    • Additional Notes: You may be able to play this game will lower quality specifications due to the simplicity of the game itself.

I had to leave the "game" open for the time shown just to write this review; I actually spent 4 minutes looking at it, going through the options ( non-existant ), etc...

The point of the game is to click your mouse button when the circle which shrinks or grows reaches the identical size of the static circle; ie a reactionary "game".


I don't recommend this as a game, however as a part of therapy in order to improve reaction time it could be beneficial because of the random location of the circle you're required to focus on different areas and guage speed before clicking.



The reason I'm not going to recommend this "game" is because it's hardly a game. It's obvious that it was meant for mobile because of the lack of options ( although there wouldn't be much of a need ); the developer could've at least spent another few moments on the code to add a sound-slider for music / effects but there's only one symbol which mutes all of it.

I could've written this game in as little as 5 minutes. It consists of a background image ( either generated or static ) whereby another background image is flashed on screen every now and again ( so the primary bg could be black with a transparency active png/gif being displayed on top and then every now and again with an offset ), a large and small circle ( random sizes in a range ) placed on screen at a random location ( for simplicity we can say top/left of a square around the circle is 0, 0 so the maximum x and y coord [ clamped ] is Screen Width - largest circle size, Screen Height - largest circle size so it always shows on screen ), then it waits for clicks and compares the current size of both circles ( or it calculates the time it'll take for the one circle to be at the exact size of the other and uses the click time as a comparitor ).

As you can see, it's a very simple system and the idea is usually integrated into games differently ( ie allowing the player time to hit a button to successfully complete a move, making sure the player is actively playing to entire a wall or other trap isn't hit [ in the case of the helicopter flash game which moves sideways all the time and the player has to press space to apply upward "thrust" to prevent collisions with the ground of varying height and a ceiling; is more complex than this game ]. Calling this a game is an injustice.

The menu is non-existent.

I purchased this as part of a bundle and I am trying to give fair reviews so the people reading this can see an honest review of this title and others I review.
