About This Software

Substance Designer is a node-based texture compositing tool that allows you to create Substance files or bitmap textures. You can use it to texture assets and also bake model information (ex: normal, displacement, curvature etc.). It contains a wide library of tools, materials and procedural effects to help you achieve your goals in a fully non-destructive workflow. It is the industry-standard for PBR texturing and is extensively used by more than 50 AAA studio game projects.

AgentWD40
( 466.9 hrs on record )
Posted: 30 December, 2015
While everyone is looking at Painter, the real power is in Designer. I started using Designer to create Micro Bump Maps. Those tiny bumps in the surface that only show up in the Specular. As I learn the software, I was amazed at how powerful it was. It's (almost) a node base Photoshop. Blur, Motion Blur, Histogram, Scale and Transform tool, Invert Grayscale, Levels, and Layers (called Blend) is all there! Plus you can see the UV on the texture map!

This program is nuts!

NOTE: This is not a perfect Photoshop recplacment. To create masks you have to create a SVG file. The Overlay and Multiply are hidden away in the Blend node options. Designer wants to tile everything, to stop this option is hidden away in the Transform node. Designer HATES non-uniform size images. All imported images need to be at x2, A 234 x 533 image will come in distorted (This happens when you attach a Transform Node to the image). The last big problem is how Designer hates layering imported images. I've layed 12 images together only to have Designer crash on me. After the 5th crash, I've had to create a new Graph, import that graph to the original graph just so Designer wouldn't crash.

If you texture models you need this program.

Go to YouTube and watch the tutorials. Digital Tutors has an amazing tutorial on Designer called "Texturing Sci-Fi Game Assets in Substance Designer." After you've watched the Youtube videos, watch this one. (Warning: Expert Mode)

A texture artist's wet dream. The learning curve isn't incredibly steep, but it takes some time to familiarize yourself with the functionality of each available node. When combining them the possibilities are pretty much unlimited. If you're familiar with Photoshop's adjustment layers and a node-based workflow you'll feel right at home. If you require more than already provided you can dive into FXMap nodes and pixel processing (again, fully node based).

The ability to expose any node's parameter to the end-user is where Substance is truly revolutionary. For instance, the amount of wear on an axe or bird â¥â¥â¥â¥ on a monument can be modified in the final Substance file, should you choose to. All of these parameters can be accessed through your game code in the supported game engines. All this packed into a few kilobytes (if you play your cards right that is).

I've used nothing but Substance Designer in the last year for the high- to low-poly baking workflow. It's extremely lightweight and bakes a set of maps in less than 10 seconds.
